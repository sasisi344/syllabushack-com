# コンテンツ量産のための情報収集リスト

Webサイト公開に向けたコンテンツ（記事）作成のため、以下の情報を収集・調査します。

## 1. IPA資格に関する最新トレンドと基礎データ
ユーザーの関心が高い「変化」や「難易度」に関する正確な情報を押さえます。

*   **シラバス変更点 (Ver 6.x / Ver 9.x):**
    *   ここ1〜2年で追加された新用語（生成AI、セキュリティ関連など）は何か？
    *   *狙い目:* まだ解説記事が少ないため、検索上位を取りやすい。
*   **直近の試験傾向:**
    *   CBT方式における「過去問流用率」の変化。
    *   最近受験した人のSNSでの感想（「ここが難しかった」「見たことない用語が出た」など）。
*   **統計データ:**
    *   合格率の推移、受験者層の変化（非エンジニアの増加など）。
    *   *記事ネタの切り口:* 「なぜ合格率が下がったのか？」などの考察。

## 2. 勉強効率化テクニック（Hack要素）
"Syllabus Hack" というサイト名にふさわしい、具体的で再現性のあるテクニックを探します。

*   **エンジニアならではのツール活用術:**
    *   ChatGPT/Geminiを使った「自分専用の解説作成」プロンプト。
    *   Notionでの学習ログ管理、Ankiアプリのデッキ作成法。
*   **科学的学習法と資格の組み合わせ:**
    *   ポモドーロ・テクニック × 過去問演習。
    *   アクティブリコール（想起練習）をIT用語暗記にどう応用するか。
*   **「隙間時間」の極意:**
    *   通勤スマホ学習だけで合格した人の具体的なアプリ・サイト活用実績。

## 3. SEOキーワードとユーザーの悩み（Yahoo!知恵袋/Googleサジェスト）
ユーザーが実際に検索している「悩み」に答える記事を作ります。

*   **ネガティブキーワード（不安の解消）:**
    *   「基本情報 意味ない」「ITパスポート 恥ずかしい」「応用情報 難しすぎる」。
    *   *記事の型:* ネガティブを否定し、メリットを提示してモチベーションに変える。
*   **具体的な疑問（How-to）:**
    *   「午前免除 申し込み方法」「プログラミング未経験 言語選択 python 表計算」。
    *   「勉強時間 足りない」「一夜漬け」。

## 4. マインドセット・キャリア論
試験勉強を続けるモチベーションとなる「未来の姿」を提示します。

*   **資格取得後のキャリアパス:**
    *   「応用情報取ったら年収上がる？」「転職に有利な推奨資格セット」。
    *   未経験からエンジニアになるロードマップにおけるIPA資格の位置づけ。
*   **挫折防止・メンタル管理:**
    *   「やる気が出ない時の対処法」「不合格だった時の立ち直り方」。
    *   「完璧主義を捨てる」6割合格のための割り切り戦略。

### 次のアクションプラン（調査実行）

このリストに基づき、以下の手順でネタを具体化します。

1.  **Web検索 & SNS検索:** `search_web` ツールを使用し、上記項目の「具体的なキーワード」や「最新の生の声」を収集する。
2.  **記事タイトル案の作成:** 収集した情報を元に、クリックしたくなる見出し（タイトル）を20〜30案リストアップする。
3.  **優先順位付け:** 「書きやすさ」と「需要」のマトリクスで、最初に書くべき5記事を決定する。
